본 프로젝트는 한국투자증권(KIS) Open API를 활용하여 미국 주식 자동매매 시스템을 구축해보는 파이썬(Python) 예제 코드입니다.  
이 코드는 증권사 API 통신 방법과 스케줄러 자동화, 텔레그램 봇 제어 등을 학습하기 위한 기술적 레퍼런스로 작성되었습니다.

## ✨ 주요 기술적 특징 (Key Features)  
💎 **집행 모드 4원화 아키텍처**: V14(오리지널 무매)와 V-REV(역추세) 각각에 대해 초안정성 'LOC 단일 타격' 방식과 시장 미시구조 추종형 'VWAP 타임 슬라이싱' 방식을 도입하여 총 4개의 독립된 퀀트 집행 모드를 지원합니다.  
💎 **수동 VWAP 시그널 모드 탑재 (수수료 방어)**: 증권사 API 수수료를 100% 방어하기 위해 봇을 시그널 참모로 전환하는 수동 모드를 신설했습니다. 자동 주문(EXEC)을 락다운하고 텔레그램 지시서를 통해 장 마감 30분 전 수동 세팅 가이던스 및 치명적 경고망을 렌더링합니다.  
💎 **V14 VWAP 유동성 추적 엔진 이식**: 오리지널 무한매수법의 강력한 T값 및 예산 공식을 유지하면서, 장 마감 30분 전(15:30 EST)부터 1분 단위 유동성 곡선(U-Curve)을 따라 예산을 분할 사냥하는 정밀 슬라이싱 플러그인을 결합했습니다.  
💎 **타임라인 시프트 및 유령 매수 차단**: VWAP 슬라이싱 타임 윈도우를 '장 마감 33분 전 ~ 4분 전(04:27~04:57 KST)'으로 앞당겨 물리적 미체결 맹점을 원천 차단했습니다. 또한 0주 졸업 달성 시 즉각 스케줄러를 동결시키는 'Daily Buy-Lock'을 이식하여 유령 매수를 100% 억제합니다.  
💎 **스윕 피니셔 거래소 락다운 우회 디커플링**: 장 마감 3분 전(15:57~16:00 EST) 스윕 덤핑 시 미국 거래소 취소 불가 규정으로 묶인 수동 긴급 수혈(MOC) 물량을 피하기 위해 KIS 실시간 원장을 재스캔합니다. 전체 수량이 아닌 '순수 매도 가능 수량(ord_psbl_qty)'만을 정밀 타격하며, 락다운 시 덤핑을 자동 생략(Skip)하는 무결점 방어막이 가동됩니다.  
💎 **핫픽스 그랜드 슬램 및 런타임 붕괴 방어**: 빈 장부 스캔 시 발생하는 런타임 붕괴 및 Windows 권한 락(Lock)을 차단하고, 토큰 만료 시간, 자정 래핑, 서머타임 프리마켓 타임존 오판 등 시간 연산 맹점을 수학적으로 완벽히 교정했습니다.  
💎 **차세대 AVWAP 하이브리드 암살자 엔진**: V-REV 본대와 100% 독립된 물량/예산 격리 방화벽을 구축했습니다. 잉여 현금 100%를 무기화하여, 듀얼 레퍼런싱(Dual-Referencing) 기반의 기초자산 팩트를 스캔하고 파생상품 호가창에 즉각 타격하는 투트랙 암살 전술이 탑재되었습니다.  
💎 **리버스 모드 절대 하드스탑 확정 탈출**: 장중 휩소(Whipsaw)에 의한 무한 스위칭 API 과부하를 원천 방어합니다. 매일 시스템 초기화 시점(17:00 KST)에 단 한 번만 TQQQ(-15%) / SOXL(-20%) 회복 여부를 팩트 스캔하여 가변 변수 의존 없이 V14 본대로 확정 복귀합니다.  
💎 **V14 대박익절 무결점 스윕 피니셔**: 대박 익절(리버스 생략) 조건 달성 시, 100% 지정가 전량 덤핑 후 즉시 루프를 탈출(Early Return)하여 단 1주의 물량도 야간 시장 하락 리스크에 노출되지 않도록 누수 버그를 영구 차단했습니다.  
💎 **V-REV 층별 고유 단가 독립 탈출 (Decoupling)**: 기존의 전체 평단가 의존 로직을 폐기하고, 각 매수 지층(Layer)이 본연의 고유 진입 단가를 기억하여 조건 충족 시 독립적으로 개별 탈출하는 후입선출(LIFO) 아키텍처를 완벽 이식했습니다.  
💎 **수동 긴급 수혈 (Emergency MOC) 시스템**: 통신 지연에 따른 자동 스케줄러의 오판 및 강제 청산 엣지 케이스를 차단했습니다. 텔레그램 큐(Queue) 관리 메뉴에서 3중 경고를 거쳐 최근 물량(LIFO)을 시장가(MOC)로 강제 매도하여 안전하게 실탄을 확보합니다.  
💎 **스마트 장부 동기화 엔진 (TrueSync)**: 미국장 영업일 스케줄을 자동으로 인식하며, 과거 데이터를 덮어쓰지 않고 오차만큼만 교정하는 비파괴 보정(CALIB) 및 야후 연동 액면분할 자동 소급 로직을 지원합니다.  
💎 **텔레그램 4단계 그랜드 리팩토링 (V28.03)**: 비대해진 telegram_bot.py를 장부 동기화/졸업 판별(telegram_sync_engine), 인라인 콜백 라우터(telegram_callbacks), 상태 머신(telegram_states), UI 렌더러(telegram_view) 4개의 독립 모듈로 완벽 분할 캡슐화하여 유지보수성과 안정성을 극대화했습니다.  
💎 **10시 정각 졸업 락온 (Time-Lock)**: 가결제 오차를 원천 차단하기 위해 0주 졸업 카드 발급 및 장부 초기화는 오직 10:00 KST 이후 단 한 번만 실행되도록 Time-Lock 방어막을 구축했습니다. Fail-Safe 발동 시 V-REV 큐 장부는 절대 소각되지 않고 보존됩니다.  
💎 **파이어게이트식 확정 정산 아키텍처 (V27.20)**: 가결제 착시와 이중 장부 엇박자를 원천 차단합니다. 봇의 '0주 스윕 즉시 졸업 카드 발급' 로직을 전면 폐기하고, 아침 증권사 서버의 수수료/세금이 100% 반영된 '실현손익 팩트 데이터'를 직접 스캔하여 오차 없는 졸업 카드를 자동 출력합니다.  
💎 **도파민 폭발 GIF 애니메이션 졸업 카드 (V27.23)**: 텔레그램 졸업 카드 발급 시 멀티 프레임 스티칭(Stitching) 기술을 도입하여 GIF 애니메이션의 모든 프레임에 수익금을 정밀 앵커링하고, send_animation API를 통해 텔레그램으로 발송하는 극한의 도파민 생성 아키텍처가 탑재되었습니다.  
💎 **시스템 자가 업데이트 엔진 (V27.00)**: 깃허브 강제 동기화(git fetch & reset) 및 GCP 데몬(systemctl) 자동 재가동을 통한 무결점 원격 패치 아키텍처와 2단계 승인 락온이 이식되었습니다.  
💎 **VWAP 앵커 고정(Lock-on) 및 영속성 듀얼 캐싱 (V28.05)**: 16:00 경계 야후 파이낸스 캐시 오염으로 인해 매 분 타점이 변동되던 엣지 케이스를 원천 차단했습니다. 당일 최초 1회만 전일 종가를 앵커로 고정하고, 서버 재시작 시 VWAP 잔차가 증발하는 기억상실 맹점을 L1(메모리)/L2(.json 파일) 원자적 동기화로 100% 영속화했습니다.  
💎 **Broker 딥-디듀플리케이션 및 분할 체결 병합 엔진 (V28.04)**: KIS API가 단일 LOC 주문을 2회 중복 반환하여 장부에 2주로 계상되던 치명적 맹점을 완벽 수술했습니다. 주문번호(odno) 기반 집계 딕셔너리로 중복 체결은 하나로 병합하고 분할 체결(Partial Fill)은 수량 누적/가중 평균하는 비파괴 아키텍처가 완비되었습니다.  
## 🛠️ 설치 및 실행 방법 (Installation & Usage)  
📌 **1. 필수 환경 (Requirements)**  
✔️ Python 3.12 이상  
✔️ 한국투자증권 Open API 발급 (App Key, App Secret)  
✔️ Telegram Bot Token 및 Chat ID  
📌 **2. 패키지 설치**  
👉 pip install requests yfinance pytz pandas_market_calendars python-dotenv pillow "python-telegram-bot[job-queue]"  
📌 **3. 환경 변수 설정 (.env 파일 생성)**  
프로젝트 최상단 폴더에 .env 파일을 만들고 아래 양식에 맞게 본인의 키를 입력합니다.  
👉 TELEGRAM_TOKEN=나의_텔레그램_봇_토큰  
👉 ADMIN_CHAT_ID=나의_텔레그램_채팅방_ID숫자  
👉 APP_KEY=나의_한국투자증권_APP_KEY  
👉 APP_SECRET=나의_한국투자증권_APP_SECRET  
👉 CANO=나의_계좌번호_앞8자리  
👉 ACNT_PRDT_CD=01 또는 22  
📌 **4. 프로그램 실행**  
👉 python main.py  
(권장: 서버 환경에서는 nohup python main.py & 명령어를 사용하여 백그라운드에서 24시간 가동되도록 설정하세요.)  
## 📂 파일 구조 (Directory Structure)  
📁 main.py : 시스템 진입점, 스케줄러 등록 및 봇 데몬 구동 코어  
📁 config.py : 전역 환경설정, JSON 데이터베이스 입출력, 장부(Ledger) 동기화 및 비파괴 보정(CALIB) 통제  
📁 broker.py : KIS Open API 및 야후 파이낸스(YF) 통신, 1분봉/현재가 파싱, 주문 전송 및 예외/타임아웃 처리  
📁 scheduler_core.py : 토큰 갱신, 자정 작업, 아침 확정 정산(10:00 KST), 리버스 확정 탈출 등 시스템 유지보수 담당  
📁 scheduler_trade.py : 정규장(17:05) 전송, VWAP 타임 슬라이싱(04:27~05:00 KST), 스나이퍼 감시, 로터리 덫 등 실전 전투 스케줄링  
📁 strategy.py : 4대 집행 모드(V14 LOC/VWAP, V-REV Auto/Manual)로 연산을 분배하고 레거시 모드를 강제 교정하는 중앙 지능형 동적 라우터(Dynamic Router)  
📁 strategy_v14.py : 오리지널 무한매수법(V14) 퀀트 공식(T값, 별값) 및 LOC 단일 타격 전술 플러그인  
📁 strategy_v14_vwap.py : V14 공식 기반 장 마감 30분 전 1분 단위 유동성 추적(VWAP) 분할 타격 전술 플러그인  
📁 strategy_reversion.py : V-REV 역추세 하이브리드 전술, LIFO 지층 분리 익절 및 공수 교대 타점 산출 코어  
📁 strategy_v_avwap.py : 차세대 AVWAP 하이브리드 암살자 전술 (듀얼 레퍼런싱 기반 기초자산 팩트 스캔 및 파생 타격)  
📁 queue_ledger.py : V-REV 전용 LIFO 로트(Lot) 장부, 팝(Pop)/푸시(Push) 및 무결성 관리  
📁 volatility_engine.py : 야후 1년 ATR 및 공포지수(VXN/HV) 동적 연산, 가중치 산출 레이더  
📁 telegram_bot.py : 텔레그램 데몬 구동 및 하위 모듈로 이벤트를 위임하는 순수 라우팅 진입점(Thin Router)  
📁 telegram_sync_engine.py : **(V28.03 신설)** 장부 동기화, 10시 정각 졸업 판별, 큐(Queue) 무결성 검증 및 비파괴 보정 비즈니스 코어  
📁 telegram_callbacks.py : **(V28.03 신설)** 인라인 키보드 클릭 이벤트 분배기 및 도메인(비상 수혈, 모드 전환 등) 전담 라우터  
📁 telegram_states.py : **(V28.03 신설)** 사용자 텍스트 입력(시드 변경, 큐 수정 등) 처리 및 팻핑거 방어 상태 기계(State Machine)  
📁 telegram_view.py : 텔레그램 UI 렌더링, 인라인 키보드 생성, 통합 지시서 포맷팅 및 졸업 이미지/GIF 드로잉  
📁 plugin_updater.py : **(V27.00 신설)** 깃허브 강제 동기화 및 GCP 데몬 자동 재가동을 통한 원격 자가 업데이트(Self-Update) 엔진  
📁 config.py : JSON 데이터베이스 입출력, 에스크로 동적 캐싱 및 수동/자동 모드 플래그 관리자  
📁 version_history.py : 코드 업데이트 최신 히스토리 기록
